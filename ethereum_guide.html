The Two-Client Architecture

Ethereum requires two pieces of software working together since The Merge (Sept 2022)

Execution Client (EL)

  • Runs the EVM (Ethereum Virtual Machine)
  • Processes transactions & smart contracts
  • Maintains account balances
  • Manages mempool (pending txs)

Examples:

Geth, Nethermind, Erigon, Besu, Reth

Consensus Client (CL)

  • Handles Proof of Stake logic
  • Manages validator committees
  • Coordinates block proposals
  • Determines chain finality

Examples:

Lighthouse, Prysm, Teku, Nimbus, Lodestar

Why both? Separation allows each layer to be optimized independently while maintaining security and decentralization.

How Blocks Are Created (Every 12 Seconds)

1

Random Selection

Every 12-second slot, one validator is randomly selected to propose the next block

2

Proposing

The proposer:

  • • Collects transactions
  • • Builds a new block
  • • Broadcasts to network
  • • Earns rewards
3

Attesting

Other validators:

  • • Verify the block
  • • Vote (attest) it's valid
  • • Sign their vote
  • • Broadcast attestation

Finality

After ⅔ of total staked ETH agrees on checkpoints (about 12-19 minutes), those blocks become final and cannot be reversed

Time Units: 1 Slot = 12 seconds | 1 Epoch = 32 slots (~6.4 minutes)

Node Types Comparison

Full Node

No ETH Required
  • • Downloads all blocks
  • • Verifies everything
  • • Serves data to apps
  • • Does NOT validate

Storage:

~700 GB - 1 TB

Validator Node

32 ETH Required
  • • Full node PLUS
  • • Proposes blocks
  • • Attests to blocks
  • • Earns staking rewards

Uptime:

>99% required

Archive Node

For Explorers
  • • ALL historical state
  • • Query any past block
  • • Deep analytics
  • • Not for normal use

Storage:

12-20+ TB (Geth)
2.5-6 TB (Erigon)

Light Client

Minimal Resources
  • • Headers + proofs only
  • • No full blocks
  • • For wallets/mobile
  • • Cannot serve data

Storage:

<1 GB

Validator Economics

Rewards

Base Staking Yield 3-4% APR

From attestations & proposals

Priority Fees (Tips) Variable

Extra from users wanting faster txs

MEV (via MEV-Boost) +20-50%

From transaction ordering

Realistic Total 4.5-5.5%

APR in ETH (not fiat)

Penalties

Inactivity Penalties

  • • Miss attestations = small penalty
  • • Extended downtime = gradual leak
  • • Recoverable with uptime

Slashing (SEVERE)

Provably malicious behavior:

  • • Double proposal (2 blocks, same slot)
  • • Surround vote (contradictory attestations)
  • • Double vote (2 attestations, same slot)

Consequences:

~1 ETH penalty + forced exit

Avoid Slashing

  • • Never run same keys on 2+ machines
  • • Use slashing protection databases
  • • Proper backup/failover procedures
